Общие сведения
EMC - Element Module Container - Многофункциональная CSS-методология. EMC не является уникальной технологией, но является попыткой усовершенствования нынешних стандартов веб-разработки
Основная CSS-парадигма - управление модулями через контейнеры, управление элементами через модули, кастомизация модулей и элементов посредством внешних и внутренних модификаторов
Element
Элемент: - переиспользуемый компонент разметки, не может содержать в себе модулей
Логический пример: обычный заголовок является элементом
Пример кода:
<h1 class="title__el">fev</h1>
Module
Модуль: - самодостаточный компонент, может включать в себя другие модули и элементы, можно встраивать в код безболезненно без дублирования кода
Логический пример: Модуль "Меню"
Пример кода:
<ul class="nav__md">
  <li class="text__el">Общие сведения</li>
  <li class="text__el">Element</li>
  <li class="text__el">Module</li>
  <li class="text__el">Container</li>
</ul>
Container
Контейнер: - внешняя "обертка" для компонентов, отвечает только за позиционирование
Логический пример: Контейнер body
Пример кода:
<body class="body__cn">
  <div class="some-module__md">
    <div class="some-element__el">1</div>
    <div class="some-element__el">2</div>
  </div>
</body>
/* containers-css */

.body__cn {
  padding: 20px;
}
Кастомизация (переопределение, внешние/внутренние модификаторы)
Переопределение: - за переопределение стилей модулей и элементов должен отвечать либо модуль высшего порядка: "HOM" либо внешние/внутренние модификаторы. В свою очередь за переопределение стилей HOM должны отвечать только внешние модификаторы. (если модификатор модуля/элемента/контейнера называется в честь одного из родителей - это называется миксин)
Модуль высшего порядка (HOM): - это модуль родительским элементом которого является контейнер
Внешний модификатор: - это модификатор который прописан в стилях модуля к которому он применен и не зависит от модуля-родителя
Внутренний модификатор: - это модификатор который прописан в стилях модуля-родителя и применяется к дочерним модулям
Пример кода:
<body class="body__cn">
  <div class="some-hom__md __wide __body__cn"> <!-- HOM может переопределять стили дочерних модулей и элементов (__body__cn - миксин) -->
    <div class="some-element__el">1</div>
    <div class="some-element__el">2</div>
    <div class="some-module__md __black __nonmargin__"> <!-- __black - Внешний модификатор, __nonmargin__ - Внутренний модификатор -->
      <div class="some-element__el">3</div>
    </div> 
  </div>
</body>
/* some-hom__md css-file */

.some-hom__md {
  width: 400px;
}
.some-hom__md.__wide {
  width: 700px;
}
.some-hom__md.__body__cn {
  text-align: center;
  font-size: 2rem;
  font-weight: 800;
}
.some-hom__md .some-module__md {
  color: yellow;
}
.some-hom__md .some-element__el {
  color: yellow;
}
.some-hom__md .some-module__md.__nonmargin__ {
  margin: 0;
}

/* some-module__md css-file */

.some-module__md {
  color: white;
  margin: 10px;
}
.some-module__md.__black {
  color: black;
}
Файловая структура и JS
Важно: - использовать модульную структуру проекта, и изолировать js файлы
Используется: - webpack для сборки бандлов (конфиг и файловая структура - https://github.com/neokkers/WebPackStarter)
Пример файловой структуры:
|- /node_modules
|- /src
  |- /_containers
    |- /indexpage
      |- _indexpage-containers.sass
      |- _indexpage-containers-media.sass
  |- /_elements
    |- _elements.sass
    |- _elements-media.sass
  |- /_modules
    |- /Nav
      |- Nav.sass
      |- Nav.js
    |- /Sidebar
      |- Sidebar.sass
      |- Sidebar.js
    |- _modules.sass
  |- /fonts
  |- /img
  |- /js 
    |- index.js
  |- /libs 
  |- /sass 
    |- _main.sass
    |- settings.sass
    |- vars.sass
  |- index.html 
  |- package.json
  |- webpack.confing.js
